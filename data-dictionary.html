Data Dictionary

AI Resume Builder — Complete Data Reference

👤 Table 1 — User
Stores authenticated user information. Managed externally by Clerk Authentication Service. The application accesses user data through Clerk's useUser() hook.
Sr. No. Field Name Data Type Size Constraints Description
1 userId String 255 PK Auto Unique identifier auto-generated by Clerk
2 firstName String 100 Required User's first name from auth provider profile
3 lastName String 100 Optional User's last name from auth provider profile
4 primaryEmailAddress String 255 Required Unique Primary email address; used to link resumes to user
5 imageUrl String (URL) 500 Optional URL to the user's profile avatar image
6 createdAt DateTime — Auto Account creation timestamp (managed by Clerk)
📄 Table 2 — Resume
Core entity storing each resume document. Persisted in Browser LocalStorage under key ai_resume_builder_resumes. Contains personal details, theme settings, and references embedded sub-collections.
Sr. No. Field Name Data Type Size Constraints Description
1 id Integer — Auto Auto-generated numeric ID via Date.now()
2 documentId String 255 PK Unique Unique document identifier; used in URL routing
3 userEmail String 255 FK → User Required Owner's email; links resume to authenticated user
4 title String 200 Required Display title of the resume (e.g., "My Tech Resume")
5 firstName String 100 Optional First name displayed on the resume
6 lastName String 100 Optional Last name displayed on the resume
7 jobTitle String 200 Optional Target job title (e.g., "Full Stack Developer")
8 email Varchar (Email) 255 Optional Regex Validated Contact email; validated with email regex pattern (e.g., "name@example.com"). Input type="email"
9 phone Varchar (Tel) 20 Optional 7-15 digits Contact phone number; validated with regex for 7-15 digits (e.g., "+91 9876543210"). Input type="tel"
10 address String 500 Optional Physical address displayed on the resume
11 themeColor String 9 Default: #7c3aed Hex color code for resume theme styling
12 summery Text 2000 Optional Professional summary paragraph (AI-generated or manual)
13 Experience Array<Object> — Default: [] Embedded array of work experience entries
14 education Array<Object> — Default: [] Embedded array of education entries
15 skills Array<Object> — Default: [] Embedded array of skill entries
16 createdAt String (ISO) 30 Auto ISO 8601 timestamp of resume creation
💼 Table 3 — Experience
Work experience entries embedded as an array inside each Resume object. Each entry represents one job position held by the user. The workSummery field can be AI-generated.
Sr. No. Field Name Data Type Size Constraints Description
1 id Integer — PK Auto Unique identifier for the experience entry
2 title String 200 Required Job title (e.g., "Full Stack Developer")
3 companyName String 200 Required Name of the employer organization
4 city String 100 Optional City of the workplace
5 state String 100 Optional State/Province of the workplace
6 startDate Date (YYYY-MM-DD) 10 Required ≤ Today Employment start date; input type="date", max constrained to current date. Displayed as "Jan 2021" on resume.
7 endDate Date (YYYY-MM-DD) 10 Optional ≥ startDate Employment end date; validated to be after startDate. Empty if currently working.
8 currentlyWorking Boolean — Default: false Whether user is currently employed at this position
9 workSummery Text (HTML) 5000 Optional Bullet-point description of responsibilities; supports HTML <ul>/<li> tags; can be AI-generated via Gemini
🎓 Table 4 — Education
Academic education entries embedded as an array inside each Resume object. Each entry represents one degree or certification earned by the user.
Sr. No. Field Name Data Type Size Constraints Description
1 id Integer — PK Auto Unique identifier for the education entry
2 universityName String 300 Required Name of the educational institution
3 degree String 100 Required Degree type (e.g., "Bachelor", "Master", "PhD")
4 major String 200 Required Field of study (e.g., "Computer Science")
5 startDate Date (YYYY-MM-DD) 10 Required ≤ Today Education start date; input type="date", max constrained to current date. Displayed as "Aug 2018" on resume.
6 endDate Date (YYYY-MM-DD) 10 Optional ≥ startDate Education end date or expected graduation; validated to be after startDate.
7 description Text 2000 Optional Additional details about the degree, coursework, or achievements
⚡ Table 5 — Skill
Professional skills embedded as an array inside each Resume object. Each entry includes a proficiency rating displayed as a visual progress bar on the resume.
Sr. No. Field Name Data Type Size Constraints Description
1 id Integer — PK Auto Unique identifier for the skill entry
2 name String 100 Required Skill name (e.g., "React", "Python", "MySQL")
3 rating Integer — Required Range: 0–5 Proficiency level (0–5 star rating); displayed as progress bar (rating × 20%) on resume preview
🗄️ Table 6 — Data Store (LocalStorage)
The application uses Browser LocalStorage as its primary data store instead of a remote database. All resume data is serialized as JSON under a single storage key.
Sr. No. Store Name Storage Key Data Format Description
1 Resume Data Store (D1) ai_resume_builder_resumes JSON Array Stores all resume objects as a JSON array. Each element is a complete Resume object with embedded Experience, Education, and Skills arrays.

⚙️ Table 8 — Process Dictionary
Documents all processes referenced in the DFD diagrams, including their input, output, and logic descriptions.
Process ID Process Name Input Output Description
1.0 User Authentication Credentials / OAuth Token Auth Session, User Profile Handles sign-in/sign-up via Clerk; manages session tokens and redirects unauthorized users
2.1 Create Resume Title, User Email New Resume Object Creates a new resume with default values (empty arrays, default theme) and stores in LocalStorage
2.2 List Resumes User Email Filtered Resume Array Retrieves all resumes from LocalStorage filtered by the authenticated user's email address
2.3 Delete Resume documentId Confirmation Removes a resume from LocalStorage by filtering out the matching documentId
3.1 Edit Personal Details Name, Job Title, Contact Info Updated Resume Updates firstName, lastName, jobTitle, email, phone, address fields
3.2 Edit Summary Manual text or AI prompt Updated Summary Updates the summery field; optionally calls Gemini AI for content generation
3.3 Edit Experience Job details or AI prompt Updated Experience Array Add/edit/remove experience entries; optionally generates work descriptions via AI
3.4 Edit Education Education details Updated Education Array Add/edit/remove education entries with degree, major, dates, and descriptions
3.5 Edit Skills Skill name and rating Updated Skills Array Add/edit/remove skills with proficiency rating (0–5 stars); skill name is required and validated
4.0 AI Content Generation Structured prompt AI-generated text Sends prompts to Google Gemini 1.5 Flash API; returns JSON (summaries) or HTML (experience). Falls back to mock responses if API key is unavailable.
5.1 Generate Preview Complete Resume Data Rendered HTML Preview Real-time rendering of resume using React ResumePreview component with selected theme color
5.2 Download PDF Rendered Preview PDF File Triggers browser's native window.print() to save/print resume as PDF
5.3 Share Resume Resume ID Shareable URL Generates a public URL using Web Share API for sharing via social media or email
